## 2026-05-19 What RAGFlow can RAG
- Harrison asked whether external pages such as Wikipedia can also be retrieved/RAGed and what RAGFlow can RAG.
- Clarified distinction: RAGFlow can retrieve from datasets/knowledge bases that have been ingested and parsed. It is not the same as live web search unless we build or use a web ingestion/crawl step.
- RAGFlow docs show supported dataset formats include PDF/DOC/DOCX/TXT/MD/MDX, CSV/XLSX/XLS, images, PPT/PPTX, and in the dataset configuration table also HTML. Its retrieval API searches specified dataset/document IDs and returns chunks using parameters like similarity threshold, vector similarity weight, top_k, rerank_id, keyword, highlight, and cross-language settings.
- Product recommendation: keep two modes in Tutor Agent: textbook RAG (stable, curated textbook/OCR dataset) and web search/live sources (Wikipedia or other web pages fetched/crawled on demand). Wikipedia can be periodically ingested into a separate RAGFlow dataset if stable retrieval is desired, but for latest/current web facts, live search/fetch should remain separate.

## 2026-05-19 RAGFlow principle and expected quality gain
- Harrison asked what RAGFlow's RAG principle is and whether adding it would make answer quality much better.
- Explained that RAGFlow improves the retrieval/context layer rather than making the generation model inherently smarter: parse documents, split into chunks, build vector + keyword indexes, retrieve semantically relevant chunks, optionally rerank, then provide traceable citations/chunks to the LLM.
- Expected quality gain: likely significant if current Tutor Agent failures come from wrong OCR pages, weak keyword matching, page-level context stuffing, or missing citations. Less improvement if failures come from OCR errors, bad teaching prompt, weak generation model, wrong course pedagogy, or poorly chunked/imported documents.
- Recommended evaluating with a small benchmark: 30-50 real student/course questions, compare current OCR keyword retrieval vs RAGFlow retrieval by hit rate, citation correctness, answer correctness, latency, and cost before fully replacing the current path.

## 2026-05-19 Semantic concept locator use case
- Harrison asked whether true RAG can search a knowledge point and locate the relevant textbook part/chapter even when the user phrases the idea differently from the textbook.
- Confirmed this is one of the biggest benefits of semantic RAG: paraphrased user concepts can map back to textbook chunks/pages/sections through embedding similarity and reranking, rather than exact keyword overlap.
- For Tutor Agent, recommended preserving metadata on every chunk: book source, chapter, sectionId, sectionTitle, page, OCR text range, figure references, and app page image URL. Then retrieval can return both answer context and a "where in the textbook" locator.
- Useful future feature: a "Find in textbook" result panel showing likely chapter/section/page, confidence, matched excerpt, and one-click open to the existing textbook overlay.

## 2026-05-19 Running RAGFlow on Windows
- Harrison said he has a Windows computer and asked whether RAGFlow can run there.
- Checked current RAGFlow and Docker docs. Answer: yes, if the Windows machine is Intel/AMD x86_64 and can run Docker Desktop with WSL2/Hyper-V. RAGFlow's quickstart officially targets x86 CPU/Nvidia GPU Docker deployment and requires CPU >= 4 cores, RAM >= 16 GB, disk >= 50 GB, Docker >= 24.0.0, and Docker Compose >= v2.26.1.
- Windows-specific caveat: it is not native Windows execution; it runs inside Docker/WSL2. Need to set `vm.max_map_count=262144` in the docker-desktop WSL distro or via `.wslconfig` to avoid Elasticsearch/Infinity startup issues.
- Recommendation: if Harrison's Windows machine has 16-32 GB RAM and x86 CPU, use it as the first RAGFlow trial host, then have the Mac/local Tutor Agent call it over LAN via HTTP.
