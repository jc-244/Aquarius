## 工作流学到的事

1. **Discovery workflow 的 schema 字段命名很重要**。S1 第一轮用
   `real: boolean` 字段，模型把它解读为模糊：到底是 "claim is real"
   还是 "refutation is real"？S2/S3 改成 `safe_to_delete: boolean`
   后语义明确。
2. **S1 discovery 还是有结果**——只是要手工 triage。schema 没问题
   的话 70/76 candidates 是因为 "default to refuted" 的保守策略，
   而不是真的全部被 refute。Manual triage 把 25 个 "truly safe"
   candidates 找出来，最终 ship 了 14 + 5。
3. **Structural ceiling 在 S3/S4 上是 0%，不是 14%**。Plan 假设错的
   时候，再多 discovery agent 也救不回来——直接 0 行。诚实报告
   优先于推 line count。
4. **`.followup-bar-row` 这种 bare-selector vs `.answer .X` 高 spec
   shadow 是 S2 唯一可找到的模式**。Discovery 工作流跑遍 ~1,400 行
   CSS 只找出这 1 个 candidate，是 honest 结果，不是 discovery 漏。
