### 新发现的非显然问题

1. **三份文档全部和代码 / CLAUDE.md 矛盾**：
   - `docs/sync-policy.md` 说 root materials/ 是 runtime → 代码偏好 workspace/
   - `docs/WINDOWS_RAG_HANDOFF.md` 用 `WINDOWS_RAG_*` 环境变量 → 代码用 `RAGFLOW_*`
   - `PROJECT_STRUCTURE.md` 没提 workspace>root fallback（CLAUDE.md 称这是"最重要的非显然行为"）
2. **跨 client/server 的逻辑重复**：`compactWhitespace` / `sortSourcesByType` / `sourceTypeRank`
   在 app.js 和 ws-bridge.js **两边都有完全相同的实现** → silent drift 风险
3. **app.js 自己内部就有重复**：`readAsDataUrl` (L3641) 和 `readFileAsDataUrl` (L1830) 是同一
   个 FileReader→dataURL helper 的两个实现
4. **`SECTION_PREVIEWS_NEW` 被自己加倍**：每条 entry 既存了 "X.Y Title" 全 key 又存了 "X.Y" 裸
   key alias → 一个本来 ~1000 行的常量被加成 ~4000 行
5. **`process.env` 每次 LLM 调用都重读 `.env`**（`getOpenAIKey` / `getOpenRouterKey` 内调
   `loadLocalEnvFile()`） → 每个请求多一次 FS 读，并发写 `.env` 还可能部分损坏
6. **ws-bridge.js 有一个死的 `/generated/` 重复路由**（L6800 永远到不了，L6717 那个先匹配）
7. **`matplotlib_gen.py` 在结构化 spec 上对 `functions[].expr` 用了 `eval()`**（剥了 `__builtins__`），
   但 spec 来源是 LLM —— 任何让用户输入流到 `functions[].expr` 的路径都成了任意代码执行面
8. **workspace-only 的运行时依赖**：`prompts/` / `new-book-section-ocr/` / `background-ocr-v3/` /
   `background-pages-split/` / `exam-priority/` / `formula-catalog/` 都只在 workspace 树里。
   有人把 workspace/materials 当 workbench 删掉就直接挂掉
9. **三个 IIFE 之间靠 `window.__ftutor*` 通信**——KC Quiz Modal IIFE (L15606-16737) ↔ v1.2.3
   fix-pack IIFE (L19840-20249)；脚本顺序一动就静默断
10. **`#learnView#learnView` 双 ID 特异性 hack 出现 9+ 次** —— cascade 已经被楔死
