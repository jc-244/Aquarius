## 2026-05-11 B.5/B.6 大章节讲解判定排查
- Harrison 问：B.5 和 B.6 为什么没有被判定成大章节生成讲解。
- 已排查 `tutor-openclaw-ui/app.js` 与 `tutor-openclaw-ui/ws-bridge.js`。
- 前端点击有 subsections 的 syllabus section 会调用 `openChapterOverviewMode(...)`，向 `/api/section` 发送 `mode: overview`；`B.5`、`B.6` 在 syllabusDataNew 中都有 subsections，所以前端理论上会进入 overview。
- 后端 overview 是否真正出现“大章节讲解”由 `shouldGenerateParentPreludeLesson(sectionId, sectionTitle, rawPages)` 与 `parent_prelude` 缓存共同决定。只有 `hasPrelude=true` 且存在/生成出 `parent_prelude` lesson 时，才会显示大章节讲解；否则只显示 overview shell + subsection cards。
- 判定函数只截取父章节标题到第一个子章节标题之间的 OCR prelude，并要求 equationCount>=3 或 compact length>=520 且 equationCount>=1 且 sentenceCount>=3。这个规则偏窄，容易把 B.5/B.6 这种标题页/正文跨页或标题后快速进入正文的章节判掉。
- 缓存命名也不同：普通大节 lesson cache 例如 `b_5/new__track:...v21.en.md` 不等于 overview 要找的 `new__parent_prelude__aquarius_visual_latex_v2.aquarius_visual_latex_v2.en.md`；当前未看到 B.5/B.6 的 parent_prelude 缓存。
- 结论：不是 syllabus 没有 subsections，而是后端 parent prelude 判定和 parent_prelude 缓存机制导致没有作为“大章节讲解”显示。
