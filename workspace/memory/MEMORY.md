## ⚠️ 最高优先级规则（每次对话必须执行）

**每次回复结束前，必须立刻把本次对话摘要 append 到 `memory/YYYY-MM-DD.md`。**
- 不管话题多简单、多短、多无聊
- 不管有没有代码操作
- 不管话题是否切换
- 回复完 → 立刻写 → 不等 → 不拖
- 这是硬规则，不是建议。违反 = 让 Harrison 失望。

**新增硬规则（2026-05-04）：每次对话都要自动写记忆，而且要写进主记忆约束里，不允许只临时记在当天摘要里后忘掉。**
- 这条规则本身必须长期保留在主记忆 `MEMORY.md`
- 之后每次对话结束时，仍然要继续 append 到当日记忆文件
- 不能因为对话很短、只是解释、只是纠错、只是确认方向，就省略记录
- 不能等用户提醒才写，必须自动执行

**新增硬规则（2026-05-04）：只要 Harrison 说“重新生成”，默认含义就是先删除当前命中的旧缓存/旧产物，再重新生成。**
- 不要默认复用旧 lesson cache
- 不要默认在旧稿上修修补补，除非 Harrison 明确说“在现有基础上改”
- 对讲解、流程图、生成产物，一律先确认“当前真正命中的文件”并优先替换它
- 如果项目里存在多个同名副本（如根目录 vs `workspace_tutor_related_full`），必须先定位前端/后端实际读取的是哪一份，再删旧重生

**新增硬规则（2026-05-04）：讲解中必须多举例子，例子是帮助学生理解的必需品，不是可有可无的装饰。**
- 不要只给定义和公式
- 每个核心知识点默认都要配至少一个小例子、近似例子、反例或 worked example
- 如果一个概念容易混淆，优先给“正确例子 + near-miss / 反例”
- 写讲解时默认问自己：学生看完这个定义，具体例子是什么？

**新增硬规则（2026-05-04）：只要某个知识点能配图，就一定要立刻配图，不要偷懒省掉。**
- 不要只在一整页最后放一张笼统总图然后假装覆盖了所有知识点
- 每个独立知识点都要单独判断：这个点能不能配一个更贴身的图、示意图、对比图或小可视化
- 如果能配，就在这个知识点出现后尽快配上，而不是拖到后面统一处理
- 例子和图都要尽量跟着概念走，减少学生上下跳读

## Tutor Agent 核心设计理念（2026-04-12 确认）

**目标用户**：中学生，懒惰、想用最短时间学完、考高分驱动。

**设计原则（所有改动必须符合）**：
1. **进度可见** ⭐ — 让用户随时看到自己在知识地图上的位置，完成一节有打勾的快感（Harrison 特别强调）
2. **极度精简** — 每次只给最核心的概念，不堆砌
3. **考试导向** — 每个知识点直接对应「考试会怎么考」
4. **即时测验** — 讲完出题，强化「我真的懂了」的感觉
5. **时间承诺** — 「5分钟搞懂 X」，用具体时间降低启动阻力
6. **最短路径** — 直接告知「期末最可能考这几道题，掌握这几个公式就够了」

## 讲解表达优先级（2026-05-04 新增硬规则）

当 Tutor Agent 为某个知识点选择“用什么形式讲最清楚”时，默认按下面顺序判断：

1. **Interactive demo / 可交互小组件**
2. **Wikimedia Commons 高解释价值图片或动图**
3. **Wikipedia page image**
4. **gptimage2 生成教学图**
5. **LaTeX 直接展示**
6. **纯文字**

这条规则的核心原则不是“有图就上图”，而是：**可视化优先，谁最能让学生一眼看懂，就优先谁。**

### 概念类型覆盖规则（比默认排序更优先）

- **definition-first / symbol-first 概念**：
  `LaTeX` 直接优先于普通图片。
  典型例子：identity matrix、unit matrix、矩阵记号、简单结构定义。

- **motion-first / process-first 概念**：
  `interactive demo` 或高解释价值动态图优先。
  典型例子：transpose、row/column 变化、矩阵乘法 conformability、变换过程。

- **teaching-clarity-first 概念**：
  如果外部参考图不够干净、不够贴题、不够考试导向，则 `gptimage2` 优先于普通静态参考图。

### 明确排除

- **教材页取图不再作为主视觉优先路径。**
- 教材/OCR 主要作为知识与内容来源，不再默认作为视觉来源。
