# 2026-06-17 · UI Friction Fix Pack v1.2.3

## Context

Harrison flagged 8 UX issues he had been ignoring while studying, then went AFK
and asked for an autonomous fix-and-deploy session. Goal: lower friction to the
bare minimum.

## What shipped — release/v1.2.3 (commit 04939da)

| # | Issue | Fix |
|---|-------|-----|
| 1 | Page flip hidden in corners, low contrast | New always-visible bottom pager `#learnExplainPager` (neobrutalist chunky chip with Prev / position / Next). Mounted inside `#learnExplainScroll`; observes lesson nav class changes via MutationObserver + 700ms heartbeat. |
| 2 | Chapter click only expanded sidebar | Patches `.syllabus-chapter` post-render (`data-ftutorChapterHook='1'`) to also call `openChapterOverviewMode` for the chapter's first section, so the main panel shows content immediately. |
| 3 | After login the main pane was empty | `MutationObserver` on `#welcomeScreen.classList` opens `sidebarSyllabusPanel` whenever the welcome screen becomes visible. |
| 4 | No "advance to next sub-sub-chapter" | New `advanceSubsection(sectionId, title, direction)` walks `syllabusData` for the next subsection, rolling across sections and chapters. Wired into the new pager's Next button, the dog-ear corner buttons (via the same handler), and the Quick Check "Next Topic" button. |
| 5 | No learned/completed indicator | New localStorage key `aquariusCompletedSubsections.v1` (stores both raw title and parsed code variants). `renderSyllabus` is wrapped so post-render it adds `.is-completed` classes and a per-chapter `<span class="chapter-progress">N/M</span>` (or "✓ done") badge. Marks fire when the user reaches the last page of a lesson or finishes the Quick Check. |
| 6 | Equations overflowed silently | Targeted CSS: `#learnView .learn-explain-body mjx-container` (and Quick Check variants) get `overflow-x:auto` with a thin scrollbar; same for tables and `<pre>`. |
| 7 | Quick Check copy/equations too small | CSS: `#kcQuestion` 18px, `.kc-option-btn` 16px, `#kcAnswer` 16px; MathJax inside scales 1.10×–1.15×. |
| 8 | Quick Check "Next Knowledge Point" no-op at end | Fixed `advanceToNextPoint`: instead of leaving the green button as an inert no-op, repurposes it to "Next Topic →" (blue, advances to next subsection) or "Back to Syllabus" (green, no next subsection) and marks the subsection completed. |

All wired through one IIFE appended at the end of `app/app.js` (window
helpers prefixed `__ftutor*`). HTML adds a single new `<div
id="learnExplainPager">` block; CSS adds one labelled fix-pack block at
the end of `style.css`.

  writing future Playwright tests should NOT use `?view=login`.

  corners, and our new bottom pager). The corners are decorative; if
  Harrison decides they're noise, set `pointer-events:none` on
  `.lecture-page-corner` and let the pager own input.

  prior cleanup — safe to delete that markup + CSS rule together.
- Welcome screen's "Ask your Tutor" hero now competes with the
  auto-opened syllabus side panel for attention. Replace the hero with
  a syllabus-grid in a future session if it feels redundant.
