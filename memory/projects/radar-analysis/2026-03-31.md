# 雷达数据分析 - 2026-03-31 进展记录

## 数据格式确认（重要！）

### int16 格式解析
- **压缩格式**：bzip2 压缩的 .bin 文件
- **存储格式**：int16 little-endian，每像素 2 字节
- **头部长**：80 字节（不含数据）
- **数据点数**：约 1,960,301 个 int16 / 文件
- **无效值标记**：
  - `-32768`（0x8000）= 无数据/空洞，约占 78.7%
  - `-1280` = 清空天空背景标记，约占 59%（在有效像素中）
- **有效值范围**：-28660 ~ 1009（在排除无效标记后）

### dBZ 换算（尚未完全确定）
- **缩放系数**：疑似 0.1（即存储值 2000 = 实际 20 dBZ）
  - 验证：贵州10个样本 max 值均为 930~1009 → dBZ ≈ 93~101（偏大，可能是缩放系数不完全是0.1）
  - 存储值 > 200 即对应 dBZ > 20（降水判定阈值）
- **待确认**：需要学姐提供数据格式文档（CINRAD/SWAN/其他）以精确定义缩放

### 数据结构
- **贵州**：年份子目录 / 事件文件夹 / CR/ .bin
- **四川/云南**：事件文件夹直接在根目录，无年份分层
- **文件名时间戳**：`RADR_CR_YYYYMMDDHHMM.bin`
- **定西**：不同格式，文件名 `Z_RADR_I_Z9932_YYYYMMDDHHMMSS_O_DOR_CAD_CAP_FMT.bin.bz2`
