# 雷达数据分析项目

## 项目概览
- **目标**: 分析硬盘中的雷达数据，整理时间序列长度
- **核心任务**: 雷达反射率 > 20 判定为降水，进行相关统计
- **创建时间**: 2026-03-30
- **状态**: 初始化中

## 数据源（云贵川冰雹数据 - 只读）
- **贵州**: `/Volumes/PortableSSD/HAinGZ/` (2,062 个 CR 事件)
- **四川**: `/Volumes/PortableSSD/HAinSC/` (297 个 CR 事件)
- **云南**: `/Volumes/PortableSSD/HAinYN/` (907 个 CR 事件)
- **总计**: ~3,266 个冰雹事件 (CR 文件夹)

## 数据格式
- **文件夹命名**: `HA_GZ_202105102150` (HA=回波, GZ=贵州, 后12位=YYYYMMDDHHMM)
- **目标子目录**: 每个事件文件夹下的 `CR/` (Composite Reflectivity 组合反射率)
- **文件格式**: `.bin` 雷达数据 (bzip2 压缩)
- **文件命名**: `RADR_CR_YYYYMMDDHHMM.bin` (10 分钟间隔)
- **忽略**: `CAP02/03/04...` 等其他仰角数据

## 输出位置（写入）
- **主目录**: `~/Desktop/学姐数据解压/云贵川/`
- 所有分析结果、图表、报告都输出到这里

## 安全约定
- ✅ **只读**分析原始数据，绝不修改/删除硬盘内容
- ✅ 所有写入操作只在桌面输出目录进行

## 分析任务
1. **数据探索**: 查看文件结构、变量、维度
2. **降水判定**: 反射率 > 20 dBZ 视为降水
3. **时间序列整理**: 
   - 统计各时段降水事件长度
   - 可能包括：降水开始/结束时间、持续时间、强度分布
4. **输出**: 统计报告 + 可视化图表

### 降水判定规则
- 反射率 > 20 dBZ 视为有降水（存储值 > 200，假设缩放0.1）
- 间隔相同（云贵川10分钟/定西5分钟）且连续的文件为一组时间序列
- 序列长度 = 每个序列包含的文件个数

### 关键发现
- 四省均以14-16文件（约2.3-2.7小时）为典型降水持续时间，合计占89.55%
- 贵州最集中（63%序列长度为14），四川最分散（最长2096文件≈349小时）
- 四川是唯一以长度15为主的省（45.25%），非14
- 定西数据间隔约5分钟，与云贵川10分钟不同
